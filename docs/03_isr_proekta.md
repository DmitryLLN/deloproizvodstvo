# Создание иерархической структуры работ по проекту

## Общая характеристика

Иерархическая структура работ проекта представляет собой результат декомпозиции общего объема работ на отдельные этапы, подэтапы и частные действия. Применительно к разработке информационной системы «Делопроизводство» иерархическая структура работ позволяет увязать содержание проекта с этапами жизненного цикла информационной системы, упорядочить выполняемые действия и обеспечить их логическую последовательность.

Для данного проекта целесообразно использовать каскадную модель жизненного цикла, поскольку в условиях учебной разработки она позволяет последовательно переходить от анализа предметной области к формированию требований, проектированию, организации репозитория, подготовке документации и завершению проекта.

## Выбранная модель жизненного цикла

В рамках проекта информационной системы «Делопроизводство» используется каскадная модель жизненного цикла. Данная модель предполагает последовательное выполнение этапов, каждый из которых опирается на результаты предыдущего этапа.

Основными этапами жизненного цикла проекта являются:

1. Инициация проекта.
2. Анализ предметной области.
3. Формирование требований.
4. Проектирование системы.
5. Организация проектного репозитория и сопровождение материалов проекта.
6. Завершение проекта.

Применение каскадной модели в данной работе объясняется тем, что она наиболее наглядно отражает логику учебного проектирования и удобна для построения иерархической структуры работ.

## Иерархическая структура работ проекта

### 1. Инициация проекта
1.1. Формулирование цели проекта.  
1.2. Определение объекта автоматизации.  
1.3. Определение области применения системы.  
1.4. Определение предполагаемых пользователей системы.  
1.5. Подготовка общего описания проекта.

### 2. Анализ предметной области
2.1. Исследование существующего порядка делопроизводства.  
2.2. Выявление основных видов документов.  
2.3. Анализ движения документов внутри организации.  
2.4. Выявление основных участников документооборота.  
2.5. Определение проблем ручного ведения делопроизводства.  
2.6. Подготовка аналитического описания предметной области.

### 3. Формирование требований к системе
3.1. Формирование функциональных требований.  
3.2. Формирование нефункциональных требований.  
3.3. Определение требований к хранению данных.  
3.4. Определение требований к пользовательским ролям.  
3.5. Подготовка исходных материалов для проектирования.

### 4. Проектирование системы
4.1. Выделение основных подсистем информационной системы.  
4.2. Проектирование структуры данных.  
4.3. Проектирование карточки документа.  
4.4. Проектирование ролей пользователей и прав доступа.  
4.5. Проектирование маршрутов движения документов.  
4.6. Проектирование отчетных форм.  
4.7. Подготовка проектных материалов.

### 5. Организация проектного репозитория
5.1. Создание удаленного репозитория проекта на GitHub.  
5.2. Клонирование репозитория на локальный компьютер.  
5.3. Настройка Git и проверка параметров пользователя.  
5.4. Создание структуры каталогов проекта.  
5.5. Добавление файлов проекта в индекс.  
5.6. Создание коммитов.  
5.7. Отправка изменений в удаленный репозиторий.  
5.8. Работа с ветками и историей изменений.

### 6. Завершение проекта
6.1. Подготовка итоговых файлов рейтинговой работы.  
6.2. Подготовка screenshot, подтверждающих выполнение работы.  
6.3. Проверка доступности публичного репозитория.  
6.4. Формирование заключения.  
6.5. Подготовка итоговой версии работы к сдаче.

## Категории пользователей системы

В проектируемой информационной системе «Делопроизводство» выделяются следующие категории пользователей:

- администратор системы;
- руководитель;
- делопроизводитель;
- исполнитель.

## Связь категорий пользователей с функциями системы

Администратор системы обеспечивает сопровождение информационной системы, ведение учетных записей и настройку прав доступа. Руководитель работает с документами на уровне согласования, утверждения и контроля исполнения. Делопроизводитель осуществляет регистрацию документов, их учет и первичную маршрутизацию. Исполнитель работает с назначенными ему документами и отражает ход выполнения поручений.

Подробное описание прав доступа категорий пользователей приведено в файле design/user_roles_and_access.md.

## Вывод

Построенная иерархическая структура работ позволяет представить проект информационной системы «Делопроизводство» как совокупность взаимосвязанных этапов, соответствующих модели жизненного цикла информационной системы. Это обеспечивает логичную организацию проектных материалов и создает основу для последовательного выполнения рейтинговой работы.
