# Основная часть по созданию и настройке Git

Публичный репозиторий проекта:  
https://github.com/DmitryLLN/deloproizvodstvo

## Общая характеристика

В рамках выполнения рейтинговой работы для сопровождения проекта информационной системы «Делопроизводство» используется распределенная система управления версиями Git. Для публикации удаленной версии проекта используется платформа GitHub. Локальный репозиторий размещается на компьютере пользователя, а удаленный репозиторий размещается в сети Интернет и доступен по публичной ссылке.

В процессе выполнения работы были освоены основные операции системы управления версиями: проверка конфигурации Git, контроль состояния файлов, добавление файлов в индекс, создание коммитов, работа с ветками, временное сохранение незавершенных изменений, просмотр истории проекта, а также взаимодействие с удаленным репозиторием.

## 1. Проверка установленной версии Git

После установки Git была выполнена проверка доступности программы в системе. Для этого использовалась команда проверки версии Git. Результат выполнения подтверждается screenshot с отображением установленной версии программы.

Команда:
git --version

## 2. Настройка Git

Для корректной фиксации истории изменений была выполнена настройка имени пользователя и адреса электронной почты. Эти параметры используются Git для указания автора коммитов. Проверка настроек выполнялась путем вывода глобальной конфигурации.

Команды:
git config --global user.name "Имя пользователя"
git config --global user.email "Адрес электронной почты"
git config --global --list

Результат выполнения подтверждается screenshot с отображением глобальных параметров конфигурации.

## 3. Работа с локальным репозиторием

В качестве локального репозитория используется папка проекта, полученная после клонирования удаленного репозитория GitHub на локальный компьютер. Такой подход обеспечивает сразу готовую связь между локальной и удаленной версиями проекта.

Для проверки состояния локального репозитория используется команда:
git status

Данная команда показывает текущую ветку, наличие измененных файлов, новых файлов и файлов, подготовленных к фиксации. Результаты выполнения команды должны быть подтверждены screenshot.

## 4. Добавление файлов в индекс

После создания файлов и каталогов проекта изменения были подготовлены к фиксации. Для этого используется команда:
git add .

Эта команда добавляет новые и измененные файлы в индекс. После ее выполнения повторно используется команда git status, позволяющая убедиться, что файлы переведены в состояние подготовки к коммиту.

## 5. Создание коммита

После индексирования файлов выполняется фиксация изменений в локальном репозитории. Для этого используется команда:
git commit -m "Комментарий к изменениям"

Коммит представляет собой зафиксированное состояние проекта на определенный момент времени. Комментарий к коммиту должен кратко и точно отражать содержание внесенных изменений. Результат выполнения команды подтверждается screenshot.

## 6. Работа с ветками

Для демонстрации технологии ветвления была создана отдельная ветка проекта. Для этого использовалась команда:
git checkout -b feature-isr

Просмотр списка веток выполнялся командой:
git branch

Использование веток позволяет выполнять отдельные изменения изолированно от основной линии проекта и демонстрирует владение современными средствами сопровождения разработки.

## 7. Временное сохранение изменений

В процессе работы была продемонстрирована команда временного сохранения незавершенных изменений:
git stash

Просмотр списка временно сохраненных изменений выполнялся командой:
git stash list

Возврат сохраненных изменений выполнялся командой:
git stash apply

Использование данной группы команд позволяет временно освободить рабочее состояние репозитория без создания промежуточного коммита.

## 8. Просмотр истории коммитов

Для просмотра истории изменений проекта использовалась команда:
git log --oneline --graph --all

Эта команда позволяет получить компактное представление истории проекта, включая сведения о последовательности коммитов и структуре ветвления.

## 9. Работа с удаленным репозиторием

Проверка связи локального репозитория с удаленным выполнялась командой:
git remote -v

Данная команда выводит адрес удаленного репозитория и подтверждает, что локальный проект связан с удаленным репозиторием GitHub.

Для отправки изменений в удаленный репозиторий использовалась команда:
git push

Для получения сведений об изменениях из удаленного репозитория использовалась команда:
git fetch

Для получения и интеграции изменений использовалась команда:
git pull

Для демонстрации получения копии удаленного репозитория может использоваться команда:
git clone https://github.com/DmitryLLN/deloproizvodstvo.git

## 10. Вывод

В результате выполнения данного раздела были освоены основные операции системы управления версиями Git, необходимые для сопровождения проекта информационной системы «Делопроизводство». Также была подтверждена корректная работа локального и удаленного репозиториев, а результаты всех ключевых этапов должны быть подтверждены соответствующими screenshot.
