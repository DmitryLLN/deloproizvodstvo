# Настройка и подключение локального репозитория к проекту

## Общая характеристика

Подключение локального репозитория к проекту информационной системы «Делопроизводство» выполняется с целью организации централизованного хранения версий файлов, фиксации изменений и публикации материалов проекта в сети Интернет. В рамках работы локальный репозиторий размещается на персональном компьютере, а удаленный репозиторий размещается на платформе GitHub и имеет публичный статус.

В данной работе использован сценарий, при котором сначала был создан публичный удаленный репозиторий проекта на GitHub, после чего он был клонирован на локальный компьютер. Такой подход позволил сразу получить связанную пару: локальный репозиторий и удаленный репозиторий проекта.

## Алгоритм подключения локального репозитория к проекту

### Шаг 1. Создание удаленного репозитория

На платформе GitHub был создан публичный репозиторий с именем deloproizvodstvo. Выбор публичного режима обеспечивает возможность просмотра проекта по сети Интернет и передачи ссылки на репозиторий в составе результата рейтинговой работы.

Результат данного этапа подтверждается screenshot страницы репозитория на GitHub.

### Шаг 2. Клонирование удаленного репозитория на локальный компьютер

После создания удаленного репозитория была выполнена операция клонирования. В результате на локальном компьютере была создана папка проекта, содержащая рабочую копию удаленного репозитория.

Команда:
git clone https://github.com/DmitryLLN/deloproizvodstvo.git

Результат выполнения данного этапа подтверждается screenshot операции клонирования либо screenshot локальной папки репозитория.

### Шаг 3. Проверка состояния локального репозитория

После перехода в каталог проекта была выполнена проверка состояния локального репозитория.

Команда:
git status

Результат выполнения данной команды подтверждает доступность репозитория, корректность активной ветки и текущее состояние файлов.

### Шаг 4. Проверка связи с удаленным репозиторием

Для подтверждения наличия связи между локальным и удаленным репозиториями использовалась команда:
git remote -v

Результат выполнения команды подтверждает, что локальный репозиторий связан с удаленным репозиторием GitHub по корректному адресу.

### Шаг 5. Создание и изменение файлов проекта

В локальной папке проекта были созданы каталоги и файлы, соответствующие структуре учебного проекта информационной системы «Делопроизводство». Изменения затронули README.md, .gitignore и набор каталогов для размещения материалов проекта.

Результат данного этапа подтверждается screenshot структуры проекта на локальном компьютере.

### Шаг 6. Подготовка изменений к фиксации

После создания и изменения файлов была выполнена подготовка изменений к фиксации.

Команда:
git add .

Результат выполнения данного этапа подтверждается screenshot команды git status после индексации файлов.

### Шаг 7. Создание коммита

После подготовки изменений был создан коммит, фиксирующий очередное состояние проекта.

Команда:
git commit -m "Комментарий к изменениям"

Результат данного этапа подтверждается screenshot с отображением выполненного коммита.

### Шаг 8. Отправка изменений в удаленный репозиторий

После создания коммита локальные изменения были отправлены в удаленный репозиторий на GitHub.

Команда:
git push

В результате выполнения этой операции изменения стали доступны в публичном репозитории проекта. Результат подтверждается screenshot страницы репозитория на GitHub с отображением новых файлов и каталогов.

### Шаг 9. Получение изменений из удаленного репозитория

Для демонстрации взаимодействия с удаленным репозиторием были использованы команды:
git fetch
git pull

Команда git fetch позволяет получить сведения об изменениях из удаленного репозитория без автоматического встраивания их в локальную рабочую копию. Команда git pull выполняет получение и интеграцию изменений в локальную ветку.

Результаты выполнения данных команд подтверждаются соответствующими screenshot.

## Значение подключения локального репозитория

Подключение локального репозитория к проекту обеспечивает:
- контроль версий файлов проекта;
- фиксацию этапов изменения документации;
- публикацию учебного проекта в сети Интернет;
- возможность резервного хранения материалов;
- наглядную демонстрацию результатов выполнения работы.

## Вывод

В результате выполнения данного раздела был создан и использован рабочий механизм взаимодействия локального репозитория проекта «Делопроизводство» с удаленным репозиторием GitHub. Это позволило обеспечить публикацию, сопровождение и сетевую доступность материалов проекта.
